Question 1

Question: ¿Cuál es la diferencia entre Internet y la World Wide Web?

Answer: El internet es la infrestructura física de servidores y ordenadores conectados a traves del mundo la cual genera una red de redes. El internet es lo que hace capaz el intercambio de información como por ejemplo en correos, transaciones y la famosa Web. La World Wide Web por su parte es una arquitectura digital operando sobre internet que permite la transmisión y representación de mutitud de tipos de datos. Es tan versatil que es de los servicios mas populares que corren sobre el internet. Un usuario solo necesita un navegador para poder compartir y recibir información. Por lo que hemos dicho, el internet es la parte solida y de hardware que funge como cimiento para que servicios de software como la WWW puedan funcionar.

Reference: Berghel, H., & Blank, D. (1999). The world wide web. In Advances in computers (pp. 179–218). https://doi.org/10.1016/s0065-2458(08)60020-0

Question 2

Question: ¿Cuáles son las partes de un URL?

Answer: La WWW fue la primera tecnología en estandarizar la Hipervinculación de cibermedios. Los Localizadores Uniformes de Recursos (URL)estan conformador por los siguientes componentes

  • Protocolo: Son las reglas establecidas para la transimisión de datos (http o https)
  • Subdominio: Sirve para separar el servicio de la WWW de otros servicios de internet como el correo electronico.
  • TLD o Dominio Superior: Indica el tipo de organización o el país de origen
  • Ruta Camino: El la ubicación exacta de un recurso en el servidor
  • Ancla: En caso de querer dirigir a una parte especifica de la misma apgina, se usa el ancla para especificar la sección especifica

Reference: ¿Qué es una URL? - Aprende desarrollo web | MDN. (2025, March 27). https://developer.mozilla.org/es/docs/Learn_web_development/Howto/Web_mechanics/What_is_a_URL

Question 3

Question: ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Answer: Los metodos o verbos HTTP son parte del protocolo HTTP, que es el lenguaje universal que hablan todos los navegadores y servidores. No pertenecer ni a HTML ni a JavaScript. Son las reglas y capacidades nativas de la propia World Wide Web

  • GET: Solicita un recurso especifico con la finalidad de solo recuperar datos. Como en esta pagina :)
  • Head: Head solicita, igual que get, datos especificos no obstante solo recupera los encabezados
  • POST: Envía un dato especifico a un servidor. Provocando un cambio de estado y registrando un nuevo recurso con un nuevo ID
  • PUT: Por otro parte, PUT envía un dato a un servidor REMPLAZANDO un dato existente.
  • DELETE:Borra un dato

Reference: Métodos de petición HTTP - HTTP | MDN. (n.d.). https://developer.mozilla.org/es/docs/Web/HTTP/Reference/Methods

Question 6

Question: Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Answer: El codigo 200 es la señal mas clara de exito en el desarrollo web. El servidor la emite al cliente para indicar que el verbo emitidos (GET,POST,PUT,etc) se a realizado con exito.

Reference: 200 OK - HTTP | MDN. (2025, July 4). https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status/200

Question 7

Question: ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Answer: El error 404 no indica que hay un error inherente en el codigo de la pagina. Este error aparece cuando el contenido de la web ya no coincide con la URL proporcionada. Esto puede ser causado por un sinfin de factores: Errores en el servidor, DNS nuevas, Cache, etc. A diferencia del error 500 que este SI indica error en codigo. El error 404 no es señal de fallo total pero si de inspección de ambiente

Reference: ¿Qué significa el error “404 not found”? (2025, November 14). IONOS Digital Guide. https://www.ionos.mx/digitalguide/paginas-web/creacion-de-paginas-web/que-significa-el-error-404-not-found/

Por ejemplo el metodo para centrar imagener CENTER (que trate de usar) a sido remplazado por un metodo en CSS el cual es mas efectivo. Question 10

Question: ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Answer: La primordial diferencia entre HTML 4 y HTML 5 es que este ultimo engloba mas tecnologías que lo hacen mas dinámico, poderoso y en especial independiente. La vieja versión de HTML 4 se apoyaba mucho de tecnologías como CSS,JS,Flash (antes),etc. Ahora HTML5 se independiza y llega a ser capaz de generar aplicaciones web con una ayuda mas auxiliar por parte de estas tecnologías externas.

Reference: Barron, B. (2025, February 14). HTML vs HTML5: Conoce las diferencias cruciales entre ellos. Kinsta®. https://kinsta.com/es/blog/html-vs-html5/
